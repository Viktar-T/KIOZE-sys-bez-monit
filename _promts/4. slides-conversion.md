# Slide Conversion Prompt - LLM-Optimized MDX Format

## Purpose

Convert lecture content into screen-friendly MDX slide presentations for OZE (Renewable Energy Systems) course using React components. Each output should be concise, focused, and ready for classroom delivery.

## Task Definition

Transform source material into structured MDX slides following the established component architecture with:

- **One clear concept per slide** (4-7 bullet points max)
- **Visual hierarchy** using semantic components
- **Instructor-ready content** with detailed teaching notes
- **Technical accuracy** for university-level engineering education

### Slide Types by Content

- `type="info"` - General informational content
- `type="tip"` - Best practices and recommendations
- `type="warning"` - Important cautions or limitations
- `type="danger"` - Critical safety or system impact information
- `type="success"` - Results, achievements, positive outcomes

### Visual Separators by Topic

- `type="default"` - Standard slide transitions
- `type="data"` - Data analysis, monitoring content
- `type="technical"` - Technical specifications, protocols
- `type="energy"` - Energy production, OZE-specific content

## Content Creation Rules

### Slide Content Guidelines

1. **One concept per slide** - Avoid cognitive overload
2. **4-7 bullet points maximum** - Keep digestible
3. **Use **bold** for key terms** - Emphasize important concepts
4. **Progressive complexity** - Build from simple to complex
5. **Technical accuracy required** - University-level precision needed
6. **Polish language** - Academic tone (B2-C1 level)

### Instructor Notes Requirements

```markdown
**Structure for Each Slide:**
- **Opening Hook** (1 sentence) - Engage attention
- **Core Content** (3-5 sentences) - Explain key concepts
- **Practical Example** (1-2 sentences) - Real-world application  
- **Key Takeaway** (1 sentence) - Main learning outcome
- **Transition** (1 sentence) - Connection to next topic

**Language Rules:**
- Natural spoken Polish for lecture delivery
- Use **bold** for all key technical terms
- Define acronyms on first use (except DC/AC)
- Include practical examples and case studies
- Add questions to prompt student reflection
```
